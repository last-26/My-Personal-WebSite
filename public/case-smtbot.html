SMTbot case study.

Discretionary trading: the rules live in your head, the results don't. The only way to systemise it is a bot that reads the same charts you do — and places orders while you sleep.

01 · Architecture

TradingView is the eyes. Python is the brain. Bybit is the hands.

Three actors, one decision loop. Pine indicators (smt_overlay + smt_oscillator) read the chart. An MCP daemon ferries those readings into Python via TradingView Desktop's debug port. Python runs the VMC planner, applies the risk manager, and routes orders to Bybit V5 over pybit. SQLite is the async journal. Pine settle status is checked every cycle — no settled bar, no trade.

03 · Problems

Three hard problems.

P1

Bridging Pine and Python without lying.

Standard webhook approaches lose half the signal. I went with TradingView Desktop in headless-debug mode (--remote-debugging-port=9222) and an MCP daemon that reads Pine output tables straight off the rendered chart. Pine settle status is checked every cycle — if the indicator hasn't finalised the bar, the cycle skips. No settled bar, no trade. The bot becomes deterministic with the chart, not despite it.

P2

Risk circuit breakers that actually fire.

Three independent breakers live in risk_manager.py: cumulative drawdown (resets daily), consecutive losses (caps hot streaks of bad luck), and a daily R-budget (forces a stop after N units of risk are spent). Each fires synchronously before order routing. The 746-case pytest suite pins every breaker's edge — stops inside the deadband, breakers triggered by stale data, demo-only paths protected from accidental live calls (BybitClient constructor refuses live unless allow_live=True is passed explicitly).

P3

Demo → live transition without surprise.

Going live isn't flipping BYBIT_DEMO=0. It's a multi-step ritual: a separate sub-account, a fresh API key with read+trade only (never withdrawal), IP whitelisting, and that explicit allow_live=True flag. There's also a Turkish ISP layer — certain CloudFront IPs that Bybit's edge sits behind get DPI-dropped at TCP-443 by some Turkish carriers. The bot probes DNS at startup and pins to a reachable edge. If you see bybit_demo_dns_pin_failed in the logs, you fight the ISP, not the bot.

04 · What I learned

Three lines I'd write on a Post-it.

  • ↪ Pine is the source of truth, Python is the executor. Don't argue with the chart — settle status exists for a reason.
  • ↪ Tests are not excessive when the bot trades on its own. 746 cases is the cost of trusting a process to lose money without you watching.
  • ↪ TR ISP egress is an architecture concern, not a config quirk. Build the failover at startup or your bot will silently die at 3am.
